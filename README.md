# モデル変換戦略サンプル (Spring Boot & Kotlin)

## プロジェクトの目的
本プロジェクトは、書籍『手を動かしてわかるクリーンアーキテクチャ』（Tom Hombergs 著）で紹介されている「境界を超える際のモデル変換戦略」を、Spring BootとKotlinを用いて実装した、ヘキサゴナルアーキテクチャでのサンプルコード集です。  
クリーンなアーキテクチャの実装において、各レイヤー（Web、ドメイン、永続化）の境界でどのようにモデルを変換すべきか、その具体的なパターンを提示することを目的としています。
「ドメインの隔離」という理想と「開発コスト」という現実のバランスをどう取るか、状況に応じた4つの戦略を比較検討できます。

## パッケージ構成とサンプル内容
実装されている各パッケージと、対応するマッピング戦略は以下の通りです。

| パッケージ名 | 採用している戦略 | 実装のポイント |
| :--- | :--- | :--- |
| `com.sample.manage.nomapping` | **モデルを変換しない** | 全層で単一の共通モデルを使い回す、スピード優先の実装。 |
| `com.sample.manage.twoway` | **双方向でのモデル変換** | 各層が独自のモデルを持ち、境界で相互に詰め替える標準的な実装。 |
| `com.sample.manage.full` | **徹底的なモデルの変換** | 各ポート（ユースケース）ごとに専用の入出力モデルを定義する、厳格な実装。 |
| `com.sample.manage.oneway` | **一方向でのモデルの変換** | ドメイン層のインターフェースを各層のモデルが実装する、結合を抑えた実装。 |

## 動作確認
各パッケージにはコントローラーが含まれており、起動後にそれぞれのエンドポイントを呼び出すことで、実際の挙動を確認いただけます。

### エンドポイント

- `GET /nomapping/users`
- `GET /twoway/users`
- `GET /full/users`
- `GET /oneway/users`
